Homelab Wrapped — Handoff

UI states, data-source reality, design tokens & motion spec for the FastAPI + Jinja build.

WRAPPED
Rendering model. The prototype is a client-side state machine; the real app is FastAPI + Jinja with per-plugin field schemas rendered dynamically. Treat every screen below as a static Jinja variant / partial — do not port the mock state transitions. Each interactive "state" (scanning, testing, error, empty) is a distinct server-rendered template or HTMX/partial swap, not a JS animation of live data.

Target stack: plain HTML/JS, no framework. Ships alongside the working reference file (Homelab Wrapped — Flow). Presentation-only behaviour (nav, count-ups, canvas backgrounds, tilt, typing, PNG export) is pure client JS — lift it as-is. Everything in the stubbed column is a server round-trip: render the resulting variant from §1 as a Jinja partial (full page or HTMX swap). No build step required.

6 · Per-plugin field schema

The "Add a service" form and the scan "credential needed" expansion are driven by each plugin's field schema. Render fields dynamically from this; the masked/secret flag decides whether a scanned service one-click-adds or expands for a credential.

# plugin field schemas (drives Add form + scan credential step)
jellyfin:
  fields:
    name:     { type: text,   required: true }
    host:     { type: url,    required: true,  placeholder: "http://192.168.1.10:8096" }
    api_key:  { type: secret, required: true,  hint: "Jellyfin → Dashboard → Admin → API Keys" }
    timezone: { type: text,   required: false, default: "UTC" }
immich:
  fields:
    name:     { type: text,   required: true }
    host:     { type: url,    required: true,  placeholder: "http://192.168.1.10:2283" }
    api_key:  { type: secret, required: true,  hint: "Immich → Account Settings → API Keys" }
    timezone: { type: text,   required: false, default: "UTC" }
csv:
  fields:
    name:     { type: text, required: true }
    path:     { type: text, required: true,  placeholder: ".csv (header row) or .json (list of objects)" }
    timezone: { type: text, required: false, default: "UTC" }

# scan Add behaviour: any field with type:secret|text-required missing → expand inline; else one-click.
# credentials persist to config.yaml on the server; nothing leaves the box.

9 · Notes for the build

  • Story = 9 server-rendered slides; nav is client-side (click / ← → / swipe). Persist current index in the URL (?c=3) so refresh/back works.
  • PNG export is client-side Canvas 2D (1080×1350) — keep it client-side; no server render needed. One layout per card type; wait for document.fonts.ready before drawing.
  • Redaction: any card flagged private:true renders "off the record" and is omitted from both the share list and the export loop — enforce server-side, not just in the UI.
  • Destructive actions (Remove connector, Purge events) must pass through the confirm modal and a server-side POST; copy already specifies that credentials are deleted from config.yaml while built recaps are kept.
  • Proxy auth toggles two things: the sign-in screen (skip) and the Account tab (collapse). Drive both from one server flag.
  • Contrast: muted mono sublines were lifted to ≥ ~4.5:1 on --hw-bg; keep new muted text at or above --hw-text-faint.
