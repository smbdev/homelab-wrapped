3 · PNG export

Both cards are drawn client-side on a <canvas> at 1080×1350 and downloaded — no server render. downloadPng(kind) takes 'summary' or 'media'; shared header ("HOMELAB WRAPPED · 2026") and footer are drawn once, then drawSummaryCard() / drawMediaCard() paint the body.

Global rule. All of the above sit under @media (prefers-reduced-motion: reduce): CSS keyframes off, RAF canvases draw one frame and stop, count-ups/typing/sparklines jump to final. The reference file implements every fallback.

6 · Build notes

  • Summary slide is chapter 9 of 10 — inserting it shifts the outro to index 9; update progress-segment count, nav bounds, and keyboard limits accordingly.
  • Sparklines and dot-matrix are self-initialising canvases: a ref callback fits DPR and starts one RAF loop, guarded by isConnected so it stops when the slide unmounts.
  • The export PNG layout is authored twice — once as the scaled DOM preview, once in canvas draw code. Treat the canvas version as the source of truth for the downloaded file.
  • Sparkline is generic: sparkline(canvas, data[], color) — filled area + stroke + end dot, min/max normalised.
